2Webページが表示されるまで（データの流れ）

みなさんがスマホやPCで Webページを見るとき、その画面は どこかから送られてきたデータを、 ブラウザが読んで組み立てて表示したものです。まずは「ページが表示されるまでに、データがどう流れるか」を見てみましょう。

  1. ① リクエスト（要求）：ブラウザのアドレス欄に URL を入れたり、リンクを押したりすると、ブラウザが Webサーバーに「この URL のファイルをください」とお願いする（URL ＝ サーバー上のファイルの住所）。
  2. ② レスポンス（応答）：Webサーバーは、そのページを作るための HTML・画像・スタイルなどのデータを、ブラウザに送り返す。
  3. ③ レンダリング（描画）：ブラウザは受け取った HTML を解釈し、文字・画像・配置を組み立てて画面に表示する。この「解釈して組み立てる」処理を レンダリングと呼ぶ。
🔗 「ページをください」＝ URL でファイルを指定すること: URL（https://… で始まる文字列）は、サーバー上にあるファイルの「住所」です。 ブラウザは「この URL のファイルをください」とサーバーに要求します。 たとえば末尾が /index.html なら、サーバーの中の index.html というファイルを要求している、ということ。 URL の末尾を省略（例：https://www.meijo-u.ac.jp/）したときは、多くのサイトで index.html が要求されます。 （だから、公開するページのファイル名は index.html にするのが基本です。）
🔁 ページを移動するたびに繰り返す: 別のページに移動するたびに、①→②→③ がもう一度行われます。Web を見るとは、この「要求して・もらって・表示する」のくり返しです。
📦 自分のPCのファイルでも同じ: 自分のPCにある index.html をダブルクリックして開くときは、サーバーとの通信は無く、PC内のファイルをブラウザが直接レンダリングします（③ だけが起こる）。§6 で体験します。
➡ 次は「データの中身」へ: では、② で送られてくる データそのものは、どんな形をしているのでしょうか。その中心が、次のセクションの HTML です。

3HTML の中身と表示のされ方（構造と属性）

② でサーバーから送られてくるデータの中心が HTML です。 前回の Markdown が「記号で構造を書く」ものだったのに対し、HTML は もっと細かく構造を指定できる書き方です。

3-1. タグで「意味」をつける（構造）

  • HTML は、文章に「ここは見出し」「ここは箇条書き」と 意味の印（タグ） をつけたもの。
  • タグは <タグ名>内容</タグ名> の形。前が 開始タグ、後ろが 終了タグ。
  • タグは 入れ子にできる（タグの中にタグ）。この入れ子の積み重ねが、ページの 構造になる。
  • ブラウザは、この構造を読み取って（§2 の ③ レンダリング）画面に表示する。

3-2. ページの骨組み（いちばん基本の形）

▼ HTML 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>はじめてのページ</title>
</head>
<body>
  <h1>こんにちは</h1>
  <p>これは本文です。</p>
</body>
</html>
  • <head>：ページの設定（文字コード charset、タブに出る title など）。画面には出ない。
  • <body>：画面に表示される中身。
  • 全体が <html> の中に <head> と <body> が入る、という 入れ子の構造になっている。

3-3. 属性で「表示のされ方・動き」が変わる

タグには、追加の情報を 属性として付けられます。属性は 開始タグの中に 属性名="値" の形で書きます。同じタグでも、付ける属性によって 表示のされ方や挙動が変わります。

<a href="https://www.meijo-u.ac.jp/">名城大学</a>
タグ名（a＝リンク） 属性名（href） 値（リンク先のURL） 内容（画面に出る文字）

属性の例（表示結果とセットで見るのは次の §4）：

タグと属性属性が変えること（表示・挙動）
<a href="...">リンク先の URL。クリックすると、そのページへ 移動する（挙動）。
<img src="..." alt="...">src＝表示する画像ファイル。alt＝画像が出ないときの代わりの文字。
<input type="text">type で入力欄の種類が変わる。text なら普通の文字入力、password なら伏せ字になる（表示・挙動）。
<タグ style="...">どのタグにも付けられる特別な属性。色・大きさ・余白などの 見た目を直接指定する（くわしくは §5）。
🧩 同じタグでも属性で別物に: たとえば <input> は、type 属性ひとつで、文字入力欄・パスワード欄・チェックボックスなどに変わります。属性は「タグの細かい指定スイッチ」だと考えるとわかりやすいです。
📖 公式の仕様（参考）: HTML Living Standard ／ 日本語訳（今は眺めるだけでOK）

4-8. 特殊な文字の表示（エスケープ）

HTML では、<・>・& の3文字は 特別な意味を持つ記号です。 < と > は タグの始まり・終わりを表し、& は次に説明する 特殊文字の書き始めを表します。

⚠ そのまま書くとどうなる？: たとえば本文に <b> とそのまま打つと、ブラウザはそれを 「太字タグ」だと解釈してしまい、文字として画面に出せません。 5 < 10 の < も「ここからタグが始まる」と勘違いされ、表示が崩れます。

そこで、これらの文字を 文字として表示したいときは、次のような 別の書き方（エスケープ／文字実体参照）に置きかえます。

表示したい文字HTML での書き方読み
<（小なり）&lt;less than
>（大なり）&gt;greater than
&（アンパサンド）&amp;ampersand

いずれも & で始まり ; で終わる形です。ブラウザはこれを読んで、対応する記号を画面に表示します。

▼ HTML（res/html_samples/08_エスケープ文字.html の中身） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>特殊な文字（エスケープ）</title>
</head>
<body>

<p>「&lt;br&gt;」は改行のためのタグです。</p>
<p>AT&amp;T のように、&amp; は「&amp;amp;」と書きます。</p>
<p>5 &lt; 10 は「5 は 10 より小さい」という意味です。</p>

</body>
</html>
表示結果（ブラウザ素の表示）
💡 なぜ必要？: HTML は「タグ（< >）で構造を組み立てる」しくみなので、構造に使う記号と、文章として見せたい記号を 区別する必要があるからです。だから「文字として見せたい記号」は、専用の書き方に置きかえます。

5スタイルの指定

タグに style 属性を付けると、色・大きさ・位置・重なりなどの「見た目」を変えられます。

▶ 各項目は クリックすると開きます。

5-1. style 属性で見た目を変える

style は、どのタグにも付けられる特別な属性で、色・大きさ・余白・枠線などの 見た目を指定します。 値は プロパティ: 値; の形で書き、セミコロン ; で区切って複数並べられます。 ここでは <button> と <div>（範囲をまとめるための箱）で、いくつかの例を見てみましょう。

▼ HTML（res/html_samples/09_style.html の中身） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>style 属性で見た目を変える</title>
</head>
<body>

<h3>ボタンの見た目を style で変える</h3>
<button>ふつうのボタン</button>
<button style="color: white; background-color: #2563eb;">青いボタン</button>
<button style="color: white; background-color: #dc2626; border-radius: 9999px;">赤い丸ボタン</button>
<button style="font-size: 20px; padding: 10px 24px;">大きいボタン</button>

<h3>div の見た目を style で変える</h3>
<div style="background-color: #fef3c7;">背景に色をつけた div</div>
<div style="color: #2563eb; font-size: 24px; font-weight: bold;">文字を大きく・青く・太字に</div>
<div style="border: 2px solid #10b981; padding: 12px;">枠線と余白をつけた div</div>
<div style="background-color: #1e293b; color: white; padding: 16px; text-align: center;">背景色・文字色・中央ぞろえ</div>

</body>
</html>
表示結果（ブラウザ素の表示）
style の書き方変わる見た目
color: white;文字の色
background-color: #2563eb;背景の色
font-size: 20px;文字の大きさ
font-weight: bold;文字の太さ（太字）

💡 実験のヒント: 色（#2563eb など）や数値（20px など）を書きかえて保存 → ブラウザで F5 すると、見た目がどう変わるか確かめられます。 （このような「見た目の指定」をまとめて書くしくみが CSS ですが、この授業では style 属性で少し触れるだけにします。）
5-2. 座標で位置を指定する

ふつう、要素は書いた順に 上から下へ並びます。これを 座標（位置）で自由に置くこともできます。

  • 置きたい要素に position: absolute; を付ける。
  • left（左から）・top（上から）の距離で位置を指定する（例：left: 20px; top: 80px;）。
  • 位置の基準にしたい 外側の箱（<div>）には position: relative; を付ける。その箱の 左上が 0 の基準になる。
▼ HTML（res/html_samples/10_座標指定.html の中身） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>座標で位置を指定する</title>
</head>
<body>

<div style="position: relative; height: 140px; border: 1px solid #cbd5e1; background: #f8fafc;">
  <div style="position: absolute; left: 20px; top: 20px; background: #fef3c7; padding: 6px;">左から 20px・上から 20px</div>
  <div style="position: absolute; left: 180px; top: 80px; background: #dbeafe; padding: 6px;">左から 180px・上から 80px</div>
</div>

</body>
</html>
表示結果（ブラウザ素の表示）
💡 重なったときの前後（どちらが上？）: 座標で要素を重ねると、前後（重なり）ができます。むずかしい指定をしなくても、 HTML で後から（下の方に）書いた要素が、前面（上）に表示されます。 前後を変えたいときは、書く順番を入れかえます。

6実際に作ってみよう

6-1. 自分のページを作る
  1. デスクトップに、自分の学籍番号の名前でフォルダを作る。
  2. エディタ（メモ帳・サクラエディタ・VS Code など）を開き、§3 の「骨組み」＋上のタグを貼り付けて書いてみる。
  3. 文字コード UTF-8 で、そのフォルダに index.html という名前で保存する。
  4. 保存した index.html を ダブルクリックしてブラウザで開く → 自分の書いた内容が表示される。
  5. 書き換えて保存 → ブラウザで F5（再読み込み）→ 変化を確認。
6-2. 同じ位置に画像を重ねて表示する

5-2 の座標指定を使うと、2枚の画像を同じ位置に重ねて表示できます。 外側の箱を position: relative; にし、画像を position: absolute; left: 0; top: 0; でそろえます。後から書いた画像が上に重なります。

▼ HTML（res/html_samples/12_画像の重ね合わせ.html の中身） 
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>同じ位置に画像を重ねる</title>
</head>
<body>

<div style="position: relative; width: 480px; height: 374px;">
  <img src="./ipanema.jpg" style="position: absolute; left: 0; top: 0;">
  <img src="./neko.png" style="position: absolute; left: 0; top: 0;">
</div>

</body>
</html>
表示結果（ブラウザ素の表示）
  1. res/html_samples の ipanema.jpg と neko.png を、自分のフォルダにコピーする。
  2. 上のコードを index.html に書いて保存 → ブラウザで開くと、2枚が重なって表示される。
  3. 2枚の <img> の 書く順番を入れかえて保存 → F5 → どちらが前面になるか確かめる。
  4. left / top の数値を変えて、重なり方を調整してみる。

※ 前面の画像が、うしろの写真の重なった部分を おおいかくします。書く順番を入れかえると、どちらが前面になるか変わります。なお、透明な部分をもつ画像（PNG）を重ねると、うしろが透けて自然に合成できます。

6-3. チャレンジ：見本のようなページを作ろう

これまでのタグとスタイルを使って、下の 見本と同じようなページ を作ってみましょう。

🖼 素材のありか: この index.html と 同じフォルダの中にある 素材/ フォルダに、背景や蝶などの画像が入っています。 必要な画像を 自分のフォルダにコピーして使いましょう。

使う技（これまでの内容）:

  • 背景になる画像を <img> で置く（§4-5）。
  • 全体を position: relative; の箱（<div>）でくくる。
  • メッセージ（文字）を position: absolute; で左上に重ねる（§5-2）。
  • 蝶などの画像を position: absolute; ＋ left/top で好きな場所に置く（§5-2・§6-2。後から書いたものが上）。

※ まったく同じでなくて大丈夫です。文章・色・画像の位置は自由に変えて、自分のページにしてみましょう。

🔮 次回の予告: 次回は、CSS と かんたんな JavaScript を使って、 ユーザーの操作（マウスを乗せる・クリックするなど）に反応して動くページに挑戦します。 たとえば下のように、置いたもの（蝶や小さなキャラクター）がマウスに反応して動き回る、といったことができるようになります。

7HTML を GitHub Pages で公開する

公開のしかたは、前回 CL-12 で Markdown を公開したときと 同じ手順です。 今回は Markdown ファイルの代わりに index.html を置くだけ。

  1. GitHub で 新しいリポジトリを作る（Public＝公開）。前回作ったものを使ってもよい。
  2. §6 で作った index.html を、そのリポジトリに置く（アップロード）。
  3. Settings → Pages で公開元を選ぶ。
  4. https://（ユーザー名）.github.io/（リポジトリ名）/ を開くと、HTML がそのまま Web ページとして表示される。
📎 Markdown との違い: HTML は そのまま Web ページとして表示されます（変換の設定がいらない）。 画像を使う場合は、index.html と同じリポジトリに画像ファイルも一緒に置くこと。
⚠ 公開＝世界中から見える: 個人情報や他人の著作物は載せないこと（CL-11 §5 の考え方と同じ）。

8まとめ・課題と締切・次回予告

今回のまとめ

  • Webページは、ブラウザが サーバーからデータを受け取り（リクエスト→レスポンス）、レンダリングして表示したもの。
  • そのデータの中心が HTML。タグの入れ子で 構造を作り、属性で表示や挙動が変わる。
  • 見出し・リスト・表・リンク・画像・動画・ボタンなど、代表的なタグを書いてみた（特殊な文字はエスケープで表示）。
  • style 属性で 見た目・座標を指定し、画像を重ねて表示してみた（後から書いたものが上）。
  • 作った index.html は、GitHub Pages で そのまま公開できる。
